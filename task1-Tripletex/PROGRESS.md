# Progress Report: Tripletex AI Accounting Agent

## 1. What we have built so far

We have built a FastAPI application that acts as an AI-powered accounting agent for Tripletex. The application exposes a `/solve` endpoint that accepts a natural language prompt and then uses a Gemini large language model to convert the prompt into one or more Tripletex API calls.

The application currently supports:
*   Parsing natural language prompts into Tripletex API calls.
*   Executing multiple API calls in sequence.
*   Handling dependencies between API calls (e.g., creating a customer and then creating an invoice for that customer).
*   Authentication with the Tripletex API using a session token.
*   Passing file content (e.g., from PDFs or images) to the LLM as context.
*   Robust error handling and logging.

## 2. Recent Changes

*   **Updated Request Format:** The `/solve` endpoint now accepts a more structured request, including a `tripletex_credentials` object and an optional list of `files`.
*   **Improved Authentication:** Switched to using `auth=("0", session_token)` for Tripletex API calls, which is more robust.
*   **File Support:** The model can now receive and process files as part of the prompt, allowing for more complex, context-aware requests.
*   **URL Bug Fix:** Corrected a bug that caused malformed API URLs by removing the `/v2` prefix from the endpoints generated by the LLM.
*   **Containerization:** Added a `Dockerfile` to containerize the FastAPI application, making it easier to deploy.
*   **Docker Ignore:** Created a `.dockerignore` file to ensure a lean and secure Docker image.

## 4. Key technical details

*   **Project ID:** `ai-nm26osl-1730`
*   **Region:** `europe-west4`
*   **Model name:** `gemini-2.5-pro-preview-03-25`
*   **API endpoints supported:**
    *   `POST /employee`: Create employee
    *   `GET /employee`: List employees
    *   `POST /customer`: Create customer
    *   `POST /product`: Create product
    *   `POST /invoice`: Create invoice
    *   `POST /invoice/{id}/:payment`: Register payment
    *   `POST /travelExpense`: Create travel expense
    *   `DELETE /travelExpense/{id}`: Delete travel expense
    *   `POST /project`: Create project
    *   `POST /department`: Create department
    *   `POST /ledger/voucher`: Create voucher/accounting entry

## 5. How to run the app locally for testing

To run the application locally for testing, follow these steps:

1.  **Install dependencies:**
    ```bash
    pip install -r requirements.txt
    ```
2.  **Run the application:**
    ```bash
    uvicorn main:app --host 0.0.0.0 --port 8080
    ```
3.  **Send requests to the `/solve` endpoint:**
    You can use a tool like `curl` or Postman to send POST requests to `http://localhost:8080/solve`. The request body should be a JSON object with the following structure:
    ```json
    {
      "prompt": "Create an invoice for the attached PDF",
      "files": [
        {
          "filename": "invoice.pdf",
          "content_base64": "...",
          "mime_type": "application/pdf"
        }
      ],
      "tripletex_credentials": {
        "base_url": "https://tx-proxy.ainm.no/v2",
        "session_token": "..."
      }
    }
    ```
